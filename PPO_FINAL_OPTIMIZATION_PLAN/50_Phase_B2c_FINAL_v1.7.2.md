# Phase B2c：速度激励 + 直线高效进给（FINAL v1.7.2）
版本日期：2025-12-30  
依赖：B2a/B2b 已建立拐角平滑与出弯恢复的稳定趋势。

---

## 1) B2c 目标（把“效率”做扎实，而不是做花）

### A1 归档要求（本阶段强制）

- **本阶段每一次 run（训练 / 评估 / 消融），无论 PASS/FAIL，都必须按 A1 归档规范生成 Run Bundle（Level-2）。**
- PASS 的前提：该 run 已形成完整 Run Bundle，且 summary.verdict=PASS。  
  （换句话说：没有归档，就不存在“通过”。）
- 失败 run 也必须归档：这是定位退化/漂移/偶发性的唯一可靠证据链。

- 直线段进给更高（速度更快），同时误差仍在容许带内
- 拐角段保持平滑且降速最小（不靠“提前很早降速”糊弄）
- 全程不退化：到达终点、无停滞、硬约束满足

---

## 2) 为什么 B2c 很容易“把 B2a 的成果抹掉”
速度激励一旦设计不当，会导致两类退化：
- 为了速度牺牲贴线（误差超带）
- 为了贴线牺牲速度（全程降速，论文不认可）

因此 B2c 必须坚持分段统计：
- 直线段：速度提升是否真实存在（并且误差不退化）
- 拐角段：速度跌落是否减少（并且平滑性保持）

---

## 3) 建议的实现边界（仍然强调 RL 主导）
- 速度相关的提升优先通过 Shaping 引导学习
- Hard-impact 的速度 cap 只能作为安全底线，并且必须可开关、可追溯
- 任何改变动作执行的滤波/平滑器都属于 Hard-impact（必须明确标注）

---

## 4) B2c 的 PASS/FAIL（建议写入 verdict）
PASS：
- 相对 baseline_ref：效率提升（时间/步数下降或速度曲线上升），且误差不退化
- 拐角平滑性不退化（B2a 主指标保持）
- 成功率不下降，stall 不上升

FAIL：
- 速度提升来自“牺牲贴线”
- 或稳定来自“全程降速”
- 或强硬规则过多导致贡献不清晰
